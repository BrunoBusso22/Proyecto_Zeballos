Gestion de calidad de software

Fundamentos

  • Calidad del software: Se refiere a qué tan bien un producto de software cumple con los requisitos especificados y satisface las expectativas del usuario. No solo importa que funcione, sino también que sea confiable, fácil de usar y mantener.
  • Métricas de calidad: Son indicadores que ayudan a medir y controlar atributos del software. Ejemplos:
    • Confiabilidad: frecuencia de fallos.
    • Usabilidad: facilidad de aprendizaje y uso.
    • Eficiencia: consumo de recursos y tiempo de respuesta.
    • Mantenibilidad: facilidad de corregir y actualizar.
    • Portabilidad: capacidad de funcionar en distintos entornos.
  • Estándares internacionales:
    • ISO 9126: propone un modelo de calidad con 6 características principales y subcaracterísticas.
    • ISO/IEC 25000 (SQuaRE): amplía y actualiza la ISO 9126, estableciendo un marco para evaluar tanto el proceso como el producto, asegurando la calidad a lo largo de todo el ciclo de vida del software.

Aseguramiento de la calidad

  • El aseguramiento busca prevenir errores y garantizar que el desarrollo cumpla con los estándares de calidad establecidos.
  • Pruebas de software: proceso de ejecutar el sistema para detectar defectos y verificar que cumpla los requisitos.

Funcionales

Verifican que el software cumpla con los requisitos especificados.

No funcionales

Evalúan rendimiento, seguridad, usabilidad, entre otros aspectos.

Niveles de pruebas

  • Unitarias

    Comprueban modulos o funciones especificas

  • Integración

    Validan que los modulos de trabajo en conjunto

  • Sistema

    REvisan el sistema completo como un todo

  • Aceptación

    El cliente valida que el software cumple las necesidades

Técnicas de pruebas

  • Caja negra: se basa en las entradas y salidas del sistema sin analizar el código interno.
  • Caja blanca: examina las estructuras internas y los caminos del código fuente.
  • Basadas en experiencia: los testers aplican su conocimiento y criterio para detectar errores.

Herramientas de calidad

Gestión de pruebas: permiten planificar y documentar casos de prueba.
Ejemplos: TestLink, Zephyr, qTest.
Seguimiento de defectos: registran, priorizan y hacen seguimiento de errores detectados.
Ejemplos: Jira, Bugzilla, MantisBT.
Análisis de calidad del código: realizan revisión estática para detectar problemas en el código fuente.
Ejemplos: SonarQube, Checkstyle, PMD.
